  上次课我们利用CPU的延时做了一个模拟的电子表，使用CPU的延时得到秒时间，计算比较复杂，如果主循环的代码改变，需要用重新计算循环次数，若用1秒的延时函数，又会减低CPU的效率，不能及时刷新数码管，造成数码管不能正常显示。这次课我们来解决这个问题。
  就好比我们人要想得到一个准确的时间，我们必须要靠手表、手机等时间工具，单片机中的CPU要获取一个准确的时间，它可以使用单片机里的提供的工具，就是定时/计数器模块。51单片机内部有两个定时/计数器，分别是定时/计数器0（记为T0），定时/计数器1（记为T1），我们只要掌握T0的使用方法，T1也就掌握了，因为他们的使用方法都差不多，所以本节我们重点介绍T0的使用，解决CPU如何利用T0得到1秒钟的时间这个问题，做出9秒倒计时实验。
  T0是集成在我们单片机内部的功能模块，看不见、摸不着，比较抽象，但我们可以形象的去理解，大家都有在自己手机上调闹钟的经历吧，而且不需要别人教，直接就会用，因为在手机上设置闹钟是看得见、摸得着的，先找到闹钟界面，然后设置时间，再启动闹钟，最后等待闹钟铃声响起，知道设置的时间到。其实使用单片机中T0的方法跟手机设置闹钟类似，首先设置T0定时时间，再启动T0，最后等待定时器时间到，只是这些过程的设置比
较抽象，首先要找到设置的特殊功能寄存器，了解每个特殊位的含义，用C51语句进行设置，下面我们按照上面所说的步骤详细介绍T0使用方法。
一、设置时间
  我们在手机上设置定时时间很容易，因为手机上显示的就是时间单位，很形象。那设置T0的定时时间也这么容易吗？我们必须要掌握T0的定时原理。T0其实是通过数数达到定时目的，T0里有两个特殊功能寄存器TH0(0X8C)和TL0(0X8A)，他们可以一起并为16位的寄存器，TH0为高8位，TL0为低8位，假设初始值为0，如图10.5.1所示：
![image-20230402202415657](./assets/image-20230402202415657.png)

这16位寄存器的特殊功能就体现在，只要T0启动，这16位寄存器会隔一段时间自动累加1，直到累加到全1，如图10.5.2所示。

![image-20230402212016040](./assets/image-20230402212016040.png)

若再累加一次，这16的寄存器就会溢出，变为0，溢出的1将会存放在内存地址0x88处的TCON寄存器中的第5位（计作TF0位），该位称为T0的溢出标志位，该位是CPU判断定时器是否定时时间到的关键标志，如图10.5.3所示。
![image-20230403000935779](./assets/image-20230403000935779.png)

通过对T0原理的理解，我们明白了，要想让T0定时一定的时间，只要根据要定时的时间t和T0累加的周期T算出需要让T0累加多少次。定时时间是已知的，T0累加的周期T是多少呢？T0的全称为定时/计数器0，也就是T0有定时和计数两个功能，T0的累加周期是由它的功能决定的：作为计数功能，它的累加频率是由P3.4引脚输入的脉冲决定的，我们会在后续的实验中详细讲解；本次实验我们选用定时功能，T0的累加频率f是单片机晶振频率（计作fosc）的1/12，本开发板单片机的晶振频率为12MHZ，那T0作为定时功能时，累加频率f为1MHZ，累加周期T为1/f，这样我们可以算出T0最大定时时间为：
`tmax=2^16*T=65536/f=65536/106=0.065536s`
也就是T0最大定时时间为0.065536秒，而本实验我们需要定时1秒，远远大于最大的定时时间，只能让T0定时多次达到，既然是多次，那次数我们一般取个整，我们可以选用让T0每次定时0.05秒，定时20次，则为1秒。接下来我们来算要想让定时器定时0.05s，定时器需要累加多少个数，我们设为x
`t=x*Tm`
`x=t/Tm=t*fosc/12=0.05*12M/12=50000`
我们如何让T0知道我们想让它累加50000个数呢？我们上面介绍到了TF0溢出标志位，T0累加到溢出时，该位被自动置为1，也就是CPU可以通过该位判断出T0是否溢出，若我们让T0从0开始数，当T0累加到50000时，我们无法判断，可以选择让T0从某个初值y开始累加，累加到溢出时刚好累加了50000次，溢出CPU是可以判断的。这个初值y很好计算，只要将最大的累加次数减去50000，y的计算过程如下：
`y=216-x=65536-50000=15536=0x3cb0`
计算出y后，我们把y的高8位赋给TH0，低8位赋给TL0，代码如下所示：
```c
TH0=0x4c；
TL0=0x00；
```
大家别忘了，我们计算到现在是有前提条件的，其中一个前提条件是我们使用的是定时功能，我们如何将T0设置为定时功能呢？另外我们公式里使用的是216，其实这里的16还有两个选择，13和8，我们如何设置为16呢？我们下面来学习定时器的一个特殊的功能寄存器TMOD，图10.5.4为TMOD寄存器各位的含义，下面我具体介绍。

//递增变换1234567890
#include"REG51.h"
#include"intrins.h"
unsigned char code display_code[]={0xc0,0xf9,0xa4,0xb0,0x99,0x92,0x82,0xf8,0x80,0x90,0xbf};
void main(void)
{ 
	char i,j=9;
	TMOD=0x01;      	//设置T0打开方式为软件启动，定时功能，16位方式
	P0=0xfe;            //选择第一个数码管
	while(1)
	{	
		P2=display_code[j];   //将j的显示码赋给P2口，数码管显示j
		for(i=0;i<20;i++)      //使用定时器定时1秒
		{
			
			TH0=0x3c;
			TL0=0xb0;
			TR0=1;
			while(TF0==0);
			TF0=0;
		}
		j=(++j)%10;          //处理下一个要显示的数
	}
}
